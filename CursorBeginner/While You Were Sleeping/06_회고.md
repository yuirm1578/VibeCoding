# 회고 — 확장 과제(과제 5)

## 확장 과제에서 어려웠던 점
### 1) Windows + 한글(유니코드) 경로에서 실행 이슈
- PowerShell/CLI 환경에서 한글 경로가 깨지면서 프로젝트 디렉터리 이동(`cd`)이 실패하거나, `npm` 실행이 루트 디렉터리로 떨어져 `package.json`을 못 찾는 문제가 발생했습니다.

### 2) Next.js 이미지/외부 리소스 설정
- 외부 이미지가 깨지는 문제는 Next.js의 이미지 최적화 설정(`next.config.js`)에서 외부 도메인 허용이 필요했습니다.

## 해결 방법
### 1) 작업 디렉터리(WorkingDirectory)를 명시하고, 실행 파일을 명확히 지정
- `Start-Process` 사용 시 `-WorkingDirectory`를 프로젝트 폴더로 지정해 `package.json` 탐색 실패를 방지했습니다.
- `npm`은 `npm.ps1`이 아닌 `npm.cmd`를 호출해 Windows에서 안정적으로 실행되게 했습니다.

### 2) 외부 이미지 도메인 허용 + Next/Image 사용
- `next.config.js`에 `images.remotePatterns`로 외부 이미지 도메인을 허용했습니다.
- 카드 이미지 렌더링은 `<img>` 대신 `next/image`로 바꿔 최적화/호환성을 개선했습니다.

## 배운 점 / 다음 개선
- 실행 환경(특히 Windows/OneDrive/한글 경로)에서 발생할 수 있는 이슈를 **초기에 체크**하는 것이 생산성을 크게 올립니다.
- UI 확장 기능(차트/비교)은 목업 데이터로 빠르게 검증하고, API가 준비되면 데이터 소스만 교체하는 구조가 확장에 유리합니다.
